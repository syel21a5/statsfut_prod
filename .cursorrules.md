## SOBRE O PROJETO
StatsFut - site de estatísticas de futebol (escanteios, cartões, gols)
- Localhost: Docker (Windows, i:\GitHub\statsfut\statsfut)
- Servidor: VPS (root@vmi3040626, /www/wwwroot/statsfut.com)
- GitHub: https://github.com/syel21a5/statsfut_prod.git

## LIGAS
- Ligas PRINCIPAIS (Bundesliga, Premier League, etc): atualizadas pelo GitHub Action (automático)
- Ligas SECUNDÁRIAS (Libertadores, Sul-Americana, Championship): atualizadas manualmente via tor_league_updater
- Deep Scrape (escanteios, cartões, gols): exportado do localhost e enviado ao servidor

## COMANDOS PRINCIPAIS
### FULL SYNC (faz tudo automaticamente)
  docker compose exec web python manage.py full_sync_league --liga 7 --push --auto-commit

### Exportar deep scrape manualmente
  docker compose exec web python manage.py export_deep_scrape --liga {ID} --output /app/deep_scrape_exports/dados_deep_scrape.json
  
### Importar no servidor
  python manage.py import_deep_scrape deep_scrape_exports/dados_deep_scrape.json
  python manage.py shell -c "from django.core.cache import cache; cache.clear()"
  sudo systemctl restart statsfut

1. Usuário quer atualizar uma liga e enviar ao servidor
2. Roda: full_sync_league --liga X --push --auto-commit
3. No servidor: git pull + import_deep_scrape
4. Se der erro de partidas não encontradas pelo api_id:
   - Usar fallback por nome+data (já implementado no import_deep_scrape.py)
   - Se ainda falhar, usar export_full_matches + import_full_matches

## RESOLUÇÃO DE ERROS DE IMPORTAÇÃO (PARTIDAS NÃO ENCONTRADAS / ERROS NO IMPORT)
Se ao rodar `import_deep_scrape` no servidor der erros do tipo `X erros (partidas não encontradas)`:
1. **O que causa isso**: Diferença na grafia do nome dos times (ex: `"FC St. Pauli"` no JSON vs `"St Pauli"` no banco da VPS) quando a partida no servidor está sem o campo `api_id` preenchido.
2. **Como diagnosticar**: Rode na VPS:
   `python manage.py debug_server_sync`
   Isso listará exatamente quais times estão falhando na correspondência de strings.
3. **Como resolver**:
   - NÃO crie times duplicados!
   - Abra o arquivo `matches/utils.py`.
   - Adicione o mapeamento do nome variante para o nome canonical no dicionário `TEAM_NAME_MAPPINGS`.
     (Exemplo: `"FC St. Pauli"`: `"St Pauli"`).
   - Faça git commit + push local e puxe no servidor.
   - Rode o `import_deep_scrape` de novo. O importador lerá o novo mapeamento via `normalize_team_name` e vinculará as partidas e gols sem erros.
