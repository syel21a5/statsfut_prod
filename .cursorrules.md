## SOBRE O PROJETO
StatsFut - site de estatísticas de futebol (escanteios, cartões, gols)
- Localhost: Docker (Windows, i:\GitHub\statsfut\statsfut)
- Servidor: VPS (root@vmi3040626, /www/wwwroot/statsfut.com)
- GitHub: https://github.com/syel21a5/statsfut_prod.git

## LIGAS
- Ligas PRINCIPAIS (Bundesliga, Premier League, etc): atualizadas pelo GitHub Action (automático)
- Ligas SECUNDÁRIAS (Libertadores, Sul-Americana, Championship): atualizadas manualmente via tor_league_updater
- Deep Scrape (escanteios, cartões, gols): exportado do localhost e enviado ao servidor

## COMANDOS PRINCIPAIS
### FULL SYNC (faz tudo automaticamente)
  docker compose exec web python manage.py full_sync_league --liga 7 --push --auto-commit

### Exportar deep scrape manualmente
  docker compose exec web python manage.py export_deep_scrape --liga {ID} --output /app/deep_scrape_exports/dados_deep_scrape.json
  
### Importar no servidor
  python manage.py import_deep_scrape deep_scrape_exports/dados_deep_scrape.json
  python manage.py shell -c "from django.core.cache import cache; cache.clear()"
  sudo systemctl restart statsfut

## RESOLUÇÃO DE ERROS DE IMPORTAÇÃO (PARTIDAS NÃO ENCONTRADAS / ERROS NO IMPORT)
Se ao rodar `import_deep_scrape` no servidor der erros do tipo `X erros (partidas não encontradas)`:
1. **O que causa isso**: Diferença na grafia do nome dos times (ex: `"FC St. Pauli"` no JSON vs `"St Pauli"` no banco da VPS) quando a partida no servidor está sem o campo `api_id` preenchido.
2. **Como diagnosticar**: Rode na VPS:
   `python manage.py debug_server_sync`
   Isso listará exatamente quais times estão falhando na correspondência de strings.
3. **Como resolver**:
   - NÃO crie times duplicados!
   - Abra o arquivo `matches/utils.py`.
   - Adicione o mapeamento do nome variante para o nome canonical no dicionário `TEAM_NAME_MAPPINGS`.
     (Exemplo: `"FC St. Pauli"`: `"St Pauli"`).
   - Faça git commit + push local e puxe no servidor.
   - Rode o `import_deep_scrape` de novo. O importador lerá o novo mapeamento via `normalize_team_name` e vinculará as partidas e gols sem erros.

## SISTEMA DE HISTÓRICO DE CONVERSAS
- Conversas anteriores são salvas em `conversations/history/` (formato JSON)
- **Sempre carregue o resumo ao iniciar novo chat** (economiza tokens):
 `python conversations/load_summary.py`
- Para filtrar por liga: `python conversations/load_summary.py --liga 7`
- **Sempre salve ao final da conversa**:
 `python conversations/save_conversation.py`
- Para listar conversas: `python conversations/list_conversations.py`
- O resumo é compacto (~50 linhas) para não gastar tokens desnecessariamente
- Mapeamentos de times são acumulados entre conversas automaticamente

## REGRAS DA PÁGINA DE PARTIDA (match_detail.html) E MOTOR DE ESTATÍSTICAS
Ao fazer qualquer modificação na página de detalhes da partida ou no motor de dados (`advanced_stats.py`), siga RIGOROSAMENTE as regras abaixo:

### 1. Motor Híbrido (Poisson + Histórico)
- O `advanced_stats.py` usa um modelo híbrido: **70% Poisson (baseado em xG/Projeções) e 30% Histórico**.
- **Cartões e Chutes:** Consideram o "Game State" (favoritismo) para inflacionar/penalizar as médias.
- **Mercados:** Dinâmicos até Over 4.5 Gols. (Nota: O Over 26.5 chutes foi aposentado).
- **Odds Justas:** Existe um template filter `fair_odd` (`matches_extras.py`) que converte % de probabilidade em Odd (fórmula: `100 / probabilidade`). Exibe no HTML com `<span class="fair-odd-badge">`.

### 2. UI e Layout Elite
- O design é "premium": fundo glassmorphism (`glass-panel`), grids personalizados (`grid-elite-2`, `grid-elite-4`), e barras de progresso modernas (`hbar-container`, `mini-widget`).
- **Alinhamento:** As alturas das colunas foram perfeitamente equilibradas. Na aba de "Gols", a Coluna 1 (barras de progresso) se iguala à Coluna 2 (Widgets de BTTS e IA Top Picks). Não quebre o alinhamento adicionando margens (`<br>`) desnecessárias.
- **Anúncios:** Foi removido o espaço de anúncio inútil acima da aba "Especiais & Combos" para otimizar o layout.

### 3. Internacionalização (i18n)
- NENHUMA palavra em português deve ficar solta ("hardcoded") no `match_detail.html`.
- Utilize `{% translate "Termo" %}` em qualquer string visível (Ex: `Over 0.5 {% translate "Gols" %}`).
- **Como atualizar traduções:** Adicione a string no dicionário dos idiomas do script `translate_po.py`. Depois, rode o script localmente e recompile as mensagens no servidor:
  `docker exec statsfut-web-1 python manage.py compilemessages -i venv`
  `docker restart statsfut-web-1`
